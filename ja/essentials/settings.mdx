すべての Mintlify サイトには、基本設定を行うための `docs.json` ファイルが必要です。以下で[プロパティ](#properties)について詳しく説明します。

<div id="properties">
  ## プロパティ
</div>

<ResponseField name="name" type="string" required>
  プロジェクト名。グローバルなタイトルとして使用されます。

  例: `mintlify`
</ResponseField>

<ResponseField name="navigation" type="Navigation[]" required>
  グループ内のすべてのページを含むグループの配列

  <Expandable title="Navigation">
    <ResponseField name="group" type="string">
      グループ名。

      例: `Settings`
    </ResponseField>

    <ResponseField name="pages" type="string[]">
      ページとして機能する Markdown ファイルへの相対パス。

      例: `["customization", "page"]`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="logo" type="string or object">
  ロゴ画像へのパス、またはライトモードとダークモード用ロゴ画像へのパスを持つオブジェクト

  <Expandable title="Logo">
    <ResponseField name="light" type="string">
      ライトモードで使用するロゴへのパス
    </ResponseField>

    <ResponseField name="dark" type="string">
      ダークモードで使用するロゴへのパス
    </ResponseField>

    <ResponseField name="href" type="string" default="/">
      ロゴをクリックしたときに遷移するリンク先
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string">
  favicon 画像へのパス
</ResponseField>

<ResponseField name="colors" type="Colors">
  グローバルテーマ用の 16 進数カラーコード

  <Expandable title="Colors">
    <ResponseField name="primary" type="string" required>
      プライマリカラー。ライトモードで、ハイライトされたコンテンツ、セクションヘッダー、アクセントなどに最もよく使用されます
    </ResponseField>

    <ResponseField name="light" type="string">
      ダークモード用のプライマリカラー。ダークモードで、ハイライトされたコンテンツ、セクションヘッダー、アクセントなどに最もよく使用されます
    </ResponseField>

    <ResponseField name="dark" type="string">
      重要なボタン用のプライマリカラー
    </ResponseField>

    <ResponseField name="background" type="object">
      ライトモードおよびダークモード両方での背景色

      <Expandable title="Object">
        <ResponseField name="light" type="string" required>
          ライトモードでの背景の 16 進数カラーコード
        </ResponseField>

        <ResponseField name="dark" type="string" required>
          ダークモードでの背景の 16 進数カラーコード
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarLinks" type="TopbarLink[]">
  トップバーに含めたいリンクの `name` と `url` の配列

  <Expandable title="TopbarLink">
    <ResponseField name="name" type="string">
      ボタンの名前。

      例: `Contact us`
    </ResponseField>

    <ResponseField name="url" type="string">
      ボタンをクリックしたときに遷移する URL。例: `https://mintlify.com/contact`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarCtaButton" type="Call to Action">
  <Expandable title="Topbar Call to Action">
    <ResponseField name="type" type={'"link" or "github"'} default="link">
      `link` はボタンを表示します。`github` は、指定した URL 先のリポジトリ情報（GitHub スター数を含む）を表示します。
    </ResponseField>

    <ResponseField name="url" type="string">
      `link` の場合: ボタンがリンクする先。

      `github` の場合: GitHub 情報を取得するリポジトリへのリンク。
    </ResponseField>

    <ResponseField name="name" type="string">
      ボタン内のテキスト。`type` が `link` の場合のみ必須です。
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="versions" type="string[]">
  バージョン名の配列。ナビゲーションバーのドロップダウンで異なるバージョンのドキュメントを表示したい場合にのみ使用します。
</ResponseField>

<ResponseField name="anchors" type="Anchor[]">
  アンカーの配列で、`icon`、`color`、`url` を含みます。

  <Expandable title="Anchor">
    <ResponseField name="icon" type="string">
      アンカーを表示するために使用される [Font Awesome](https://fontawesome.com/search?s=brands%2Cduotone) アイコン。

      例: `comments`
    </ResponseField>

    <ResponseField name="name" type="string">
      アンカーラベルの名前。

      例: `Community`
    </ResponseField>

    <ResponseField name="url" type="string">
      アンカーに含めるページを判定する URL の先頭部分。通常は、ページを配置するフォルダー名になります。
    </ResponseField>

    <ResponseField name="color" type="string">
      アンカーアイコンの背景の 16 進カラーコード。`from` と `to` というプロパティを持つオブジェクトを渡すことで、それぞれ 16 進カラーコードとしてグラデーションを指定することもできます。
    </ResponseField>

    <ResponseField name="version" type="string">
      対応するドキュメントバージョンが選択されるまでアンカーを非表示にしておきたい場合に使用します。
    </ResponseField>

    <ResponseField name="isDefaultHidden" type="boolean" default="false">
      アンカー内のドキュメントに直接リンクするまでアンカーを非表示にしておきたい場合は `true` を指定します。
    </ResponseField>

    <ResponseField name="iconType" default="duotone" type="string">
      次のいずれか：&quot;brands&quot;、&quot;duotone&quot;、&quot;light&quot;、&quot;sharp-solid&quot;、&quot;solid&quot;、&quot;thin&quot;
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topAnchor" type="Object">
  最上位アンカーのデフォルト設定を上書きします。

  <Expandable title="Object">
    <ResponseField name="name" default="Documentation" type="string">
      最上位アンカーの名前。
    </ResponseField>

    <ResponseField name="icon" default="book-open" type="string">
      Font Awesome アイコン。
    </ResponseField>

    <ResponseField name="iconType" default="duotone" type="string">
      次のいずれか：&quot;brands&quot;、&quot;duotone&quot;、&quot;light&quot;、&quot;sharp-solid&quot;、&quot;solid&quot;、&quot;thin&quot;
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="tabs" type="Tabs[]">
  ナビゲーションタブの配列。

  <Expandable title="Tabs">
    <ResponseField name="name" type="string">
      タブラベルの名前。
    </ResponseField>

    <ResponseField name="url" type="string">
      タブに含めるページを判定する URL の先頭部分。通常は、ページを配置するフォルダー名になります。
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="api" type="API">
  API 設定の構成です。API ページの詳細は [API Components](/ja/api-playground/demo) を参照してください。

  <Expandable title="API">
    <ResponseField name="baseUrl" type="string">
      すべての API エンドポイントに対するベース URL です。`baseUrl` が配列の場合、ユーザーが切り替え可能な複数のベース URL オプションを有効にします。
    </ResponseField>

    <ResponseField name="auth" type="Auth">
      <Expandable title="Auth">
        <ResponseField name="method" type="&#x22;bearer&#x22; | &#x22;basic&#x22; | &#x22;key&#x22;">
          すべての API エンドポイントで使用される認証方式です。
        </ResponseField>

        <ResponseField name="name" type="string">
          API playground で使用される認証パラメータ名です。

          method が `basic` の場合、形式は `[usernameName]:[passwordName]` とする必要があります。
        </ResponseField>

        <ResponseField name="inputPrefix" type="string">
          認証入力フィールドの接頭辞として使用されることを想定したデフォルト値です。

          例: `inputPrefix` が `AuthKey` の場合、認証フィールドのデフォルト入力値として `AuthKey` が引き継がれます。
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="playground" type="Playground">
      API playground の設定です。

      <Expandable title="Playground">
        <ResponseField name="mode" default="show" type="&#x22;show&#x22; | &#x22;simple&#x22; | &#x22;hide&#x22;">
          playground を表示するか、非表示にするか、あるいは追加のユーザー操作なしでエンドポイントのみを表示する（`simple`）かを指定します。

          詳細は [playground guides](/ja/api-playground/demo) を参照してください。
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="maintainOrder" type="boolean">
      このフラグを有効にすると、OpenAPI ページ内のキーの並び順が、OpenAPI ファイルで定義されているキーの並び順と一致するようになります。

      <Warning>この挙動は近日中にデフォルトで有効になり、その時点でこのフィールドは非推奨になります。</Warning>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="openapi" type="string | string[]">
  OpenAPI ファイルを指す URL または相対パスの文字列、もしくはそれらの文字列配列です。

  例:

  <CodeGroup>
    ```json 絶対パス
    "openapi": "https://example.com/openapi.json"
    ```

    ```json 相対パス
    "openapi": "/openapi.json"
    ```

    ```json 複数
    "openapi": ["https://example.com/openapi1.json", "/openapi2.json", "/openapi3.json"]
    ```
  </CodeGroup>
</ResponseField>

<ResponseField name="footerSocials" type="FooterSocials">
  SNS アカウントを表すオブジェクトで、key:property の組が SNS プラットフォームとそのアカウント url を表します。

  例:

  ```json
  {
    "x": "https://x.com/mintlify",
    "website": "https://mintlify.com"
  }
  ```

  <Expandable title="FooterSocials">
    <ResponseField name="[key]" type="string">
      次のいずれかの値です: `website`, `facebook`, `x`, `discord`, `slack`, `github`, `linkedin`, `instagram`, `hacker-news`

      例: `x`
    </ResponseField>

    <ResponseField name="property" type="string">
      SNS プラットフォームへの URL です。

      例: `https://x.com/mintlify`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="feedback" type="Feedback">
  フィードバックボタンを有効にするための構成です。

  <Expandable title="Feedback">
    <ResponseField name="suggestEdit" type="boolean" default="false">
      ユーザーが pull request を通じて編集の提案を行えるボタンを有効にします。
    </ResponseField>

    <ResponseField name="raiseIssue" type="boolean" default="false">
      ユーザーがドキュメントに関する issue を起票できるボタンを有効にします。
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="modeToggle" type="ModeToggle">
  ダークモードのトグルをカスタマイズします。

  <Expandable title="ModeToggle">
    <ResponseField name="default" type={'"light" or "dark"'}>
      新規ユーザーに対して常にライトモードまたはダークモードを表示したい場合に設定します。設定しない場合は、ユーザーのオペレーティングシステムと同じモードがデフォルトとして使用されます。
    </ResponseField>

    <ResponseField name="isHidden" type="boolean" default="false">
      ダーク／ライトモードのトグルを非表示にするには true に設定します。`isHidden` を `default` と組み合わせて使用することで、ドキュメントをライトモードまたはダークモードのみに固定できます。例えば:

      <CodeGroup>
        ```json ダークモードのみ
        "modeToggle": {
          "default": "dark",
          "isHidden": true
        }
        ```

        ```json ライトモードのみ
        "modeToggle": {
          "default": "light",
          "isHidden": true
        }
        ```
      </CodeGroup>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="backgroundImage" type="string">
  すべてのページの背後に表示される背景画像です。[Infisical](https://infisical.com/docs) と [FRPC](https://frpc.io) の例を参照してください。
</ResponseField>
