こちらの [Benzinga Cloud](https://cloud.benzinga.com/) 開発者向けドキュメントポータルでは、ニュース、市場、企業データ向け API のドキュメントをご覧いただけます。ご不明な点がございましたら、[こちらからお問い合わせください](mailto:partners@benzinga.com)。喜んでサポートいたします。

<div id="about-this-documentation">
  ## このドキュメントについて
</div>

当社の REST API に関するドキュメントは、OpenAPI 仕様に基づいて作成されています。生の yaml 仕様は、このドキュメントアプリケーションの公開リポジトリから参照できます。[こちら](https://github.com/Benzinga/doc-site-mintlify)をご覧ください。

統合された REST クライアントが、OpenAPI ベースの各ページの「Playground」セクションに直接埋め込まれています。API キーを取得すれば、ドキュメントポータルから離れることなくリクエストをテストできます。あるいは、他の GUI ベースの REST クライアント（[Postman](https://www.getpostman.com/)、[Insomnia](https://insomnia.rest/)、[Paw](https://paw.cloud/)）や cUrl1 のような CLI ツールを使ってテストすることもできます。

最後に、OpenAPI 仕様は Swagger エコシステムの各種ツールと互換性があるため、[任意のプログラミング言語向けのクライアントを生成することも可能です](http://swagger.io/swagger-codegen/)。

<Info>
    当社のドキュメントは AI 対応です。Ctrl + K（PC）または Cmd + K（Mac）で質問してみてください。
</Info>

<div id="acessing-the-data">
  ## データへのアクセス
</div>

<Steps>
    <Step title="アカウントにログインする">
        「[Get Started](https://www.benzinga.com/apis/licensing/register)」をクリックしてアカウントを作成します。

    <Step title="APIキーをコピーする">
        ユーザーダッシュボードにログインしたら、APIキーをコピーします。

    <Step title="構築を開始！">

    </Step>
</Steps>

<div id="best-practices">
  ## ベストプラクティス
</div>

可能な場合は、Benzinga が提供するクライアントの利用をお勧めします。多くのエンドポイントは、[Python ライブラリ](https://github.com/Benzinga/benzinga-python-client)経由で利用できます。また、公開している [GitHub](https://github.com/benzinga) リポジトリでサンプル実装およびクライアントライブラリを提供しています。

最後に、特にリアルタイム取り込み（過去データの取り込みではなく）の場合には、Calendar、Signals、News API を利用する際に差分（デルタ）を活用することを強く推奨します。これは、Calendar および Signals API では `parameters[updated]` クエリパラメータを、News API では `updatedSince` クエリパラメータを使用することで実現できます。これらのパラメータを使用することで、レイテンシーを最小限に抑えることができます。
