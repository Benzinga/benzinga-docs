<div id="introduction">
  ## はじめに
</div>

Benzinga Logos API は、企業、ファンド、暗号資産の高品質なロゴにアクセスできる API です。シンボル、CIK 番号、ISIN（国際証券識別番号）コード、CUSIP、FIGI など、複数の識別子の種類を使ってロゴを検索できます。

<div id="available-endpoints">
  ## 利用可能なエンドポイント
</div>

<div id="logossearch">
  ### `/logos/search`
</div>

各種識別子を使用してロゴを検索します。ロゴのURL、証券の詳細、およびメタデータを返します。

<div id="logossync">
  ### `/logos/sync`
</div>

複数のロゴをページネーション付きで取得するための一括同期エンドポイントです。

<div id="search-filters">
  ## 検索フィルター
</div>

API は、次の 2 つの主なパラメータによる柔軟な検索をサポートしています。

- `search_keys` - 検索対象とする識別子の値
- `search_keys_type` - 識別子の種類（省略可能。省略時は `symbol` が既定値）

<div id="1-filter-by-symbol">
  ### 1. シンボルでフィルタリングする
</div>

株式のティッカーシンボルを使ってロゴを検索します。

**パラメータ:**

* `search_keys`: シンボル（例: `AAPL`, `TSLA`, `MSFT`）
* `search_keys_type`: `symbol`（省略可。指定しない場合のデフォルト値）

**例:**

```
GET /api/v2/logos/search?search_keys=AAPL&search_keys_type=symbol&fields=logo_light,logo_dark
```


<div id="2-filter-by-cik">
  ### 2. CIK でフィルタリングする
</div>

SEC の Central Index Key（CIK）を使って検索します。ゼロ埋めあり／なし両方の形式の値をサポートします。

**パラメータ:**

* `search_keys`: CIK 番号（例: `320193` または `0000320193`）
* `search_keys_type`: `cik`

**例:**

```
GET /api/v2/logos/search?search_keys=320193&search_keys_type=cik&fields=logo_light,logo_dark
```

**ゼロ埋めされた CIK の例：**

```
GET /api/v2/logos/search?search_keys=0000320193&search_keys_type=cik&fields=logo_light,logo_dark
```


<div id="3-filter-by-isin">
  ### 3. ISIN でフィルタリングする
</div>

国際証券識別番号（ISIN）を使用して検索します。

**パラメータ:**

* `search_keys`: ISIN コード（例: `US0378331005`）
* `search_keys_type`: `isin`

**例:**

```
GET /api/v2/logos/search?search_keys=US0378331005&search_keys_type=isin&fields=logo_light,logo_dark
```


<div id="4-filter-by-figi-share-class">
  ### 4. FIGI株式クラスでフィルタリング
</div>

BloombergのFinancial Instrument Global Identifier（FIGI）の株式クラスを使用して検索します。

**パラメータ:**

* `search_keys`: FIGI株式クラスID（例: `BBG001SQKGD7`）
* `search_keys_type`: `figi_share_class`

**例:**

```
GET /api/v2/logos/search?search_keys=BBG001SQKGD7&search_keys_type=figi_share_class&fields=logo_light,logo_dark
```


<div id="5-filter-by-cusip">
  ### 5. CUSIP で絞り込み
</div>

Committee on Uniform Securities Identification Procedures（CUSIP）番号を使用して検索します。

**パラメータ:**

* `search_keys`: CUSIP 番号（例: `88160R101`）
* `search_keys_type`: `cusip`

**例:**

```
GET /api/v2/logos/search?search_keys=88160R101&search_keys_type=cusip&fields=logo_light,logo_dark
```


<div id="6-filter-by-symbol-with-currency-country">
  ### 6. シンボルを通貨・国コードでフィルタする
</div>

コロン（`:`）またはピリオド（`.`）を区切り文字として使用して、シンボルに国コードまたは通貨コードを付与します。

**パラメータ:**

* `search_keys`: 修飾子付きシンボル（例: `AAPL:US`, `AAPL:USD`, `AAPL.US`）
* `search_keys_type`: `symbol`

**例:**

```
GET /api/v2/logos/search?search_keys=AAPL:US&search_keys_type=symbol&fields=logo_light,logo_dark
GET /api/v2/logos/search?search_keys=AAPL:USD&search_keys_type=symbol&fields=logo_light,logo_dark
GET /api/v2/logos/search?search_keys=AAPL.US&search_keys_type=symbol&fields=logo_light,logo_dark
```


<div id="7-filter-by-exchange-and-symbol">
  ### 7. 取引所とシンボルでフィルタリング
</div>

取引所コードとシンボルを組み合わせて検索します。Bloomberg スタイル（レガシー）と LSEG スタイルの両方の形式をサポートしています。

**Bloomberg スタイルのマッピング（レガシー）：**

* `search_keys`: 取引所コードとシンボル（例: `NASDAQ:AAPL`）
* `search_keys_type`: `symbol`

**LSEG 3 文字コード（現行）：**

* `search_keys`: 取引所コードとシンボル（例: `NSQ:AAPL`）
* `search_keys_type`: `symbol`

**例:**

```
GET /api/v2/logos/search?search_keys=NASDAQ:AAPL&search_keys_type=symbol&fields=logo_light,logo_dark
GET /api/v2/logos/search?search_keys=NSQ:AAPL&search_keys_type=symbol&fields=logo_light,logo_dark
```


<div id="8-filter-by-iso-mic-and-symbol">
  ### 8. ISO MIC とシンボルでフィルター
</div>

ISO MIC（Market Identifier Code）とシンボルの組み合わせで検索します。

**パラメータ:**

* `search_keys`: MIC コードとシンボル（例: `XYNS:AAPL`）
* `search_keys_type`: `symbol`

**例:**

```
GET /api/v2/logos/search?search_keys=XYNS:AAPL&search_keys_type=symbol&fields=logo_light,logo_dark
```


<div id="9-crypto-logo-filters">
  ### 9. 暗号資産ロゴのフィルター
</div>

さまざまな形式で暗号資産のロゴを検索できます。`search_keys_type` が指定されていない場合でも、API は自動的に暗号資産形式を認識します。

**サポートされている形式:**

* `CRYPTO:BTC`
* `CRYPTO/BTC`
* `$BTC`
* `BTC/USD`

**例:**

```
GET /api/v2/logos/search?search_keys=CRYPTO:BTC&fields=logo_light,logo_dark
GET /api/v2/logos/search?search_keys=CRYPTO/BTC&fields=logo_light,logo_dark
GET /api/v2/logos/search?search_keys=$BTC&fields=logo_light,logo_dark
GET /api/v2/logos/search?search_keys=BTC/USD&fields=logo_light,logo_dark
```


<div id="available-fields">
  ## 利用可能なフィールド
</div>

`fields` パラメータを使用して、特定のロゴ種別をリクエストできます。利用可能なフィールドは次のとおりです。

* `logo_light` - ライトテーマのロゴ (PNG)
* `logo_dark` - ダークテーマのロゴ (PNG)
* `logo_vector_light` - ライトテーマのベクターロゴ (SVG)
* `logo_vector_dark` - ダークテーマのベクターロゴ (SVG)
* `mark_light` - ライトテーマのマーク/アイコン (PNG)
* `mark_dark` - ダークテーマのマーク/アイコン (PNG)
* `mark_vector_light` - ライトテーマのベクターマーク (SVG)
* `mark_vector_dark` - ダークテーマのベクターマーク (SVG)
* `mark_composite_light` - ライトテーマの合成マーク (PNG)
* `mark_composite_dark` - ダークテーマの合成マーク (PNG)
* `mark_vector_composite_light` - ライトテーマのベクター形式の合成マーク (SVG)
* `mark_vector_composite_dark` - ライトテーマのベクター形式の合成マーク (SVG)

**複数のフィールドをリクエストする例:**

```
GET /api/v2/logos/search?search_keys=TSLA&fields=logo_light,logo_dark,mark_vector_composite_light
```


<div id="additional-parameters">
  ## 追加パラメータ
</div>

<div id="image-customization">
  ### 画像カスタマイズ
</div>

- `scale` - 画像のスケール（例：`100x100`、`300x300`）
- `max_width` - レスポンスで返される画像の最大幅
- `composite_radius` - 合成画像の角丸半径（整数、`0-50` の範囲）
- `composite_auto` - 合成画像を自動生成するかどうかを指定（真偽値）

<div id="pagination-bulk-sync-only">
  ### ページネーション（バルク同期専用）
</div>

- `page` - ページ番号（整数）
- `pagesize` - 1ページあたりの結果数（整数）
- `updated_since` - 更新日時によるフィルタリング（ISO 8601 形式）

<div id="security-details">
  ### 証券の詳細
</div>

- `securities` - レスポンスに証券の詳細情報を含めるかどうかを指定します（boolean）

<div id="authentication">
  ## 認証
</div>

すべての Logos API エンドポイントは、API キーによる認証が必要です。クエリパラメータとしてトークンを指定してください。

```
?token=YOUR_API_KEY
```


<div id="best-practices">
  ## ベストプラクティス
</div>

1. **search_keys_type を指定する** - API のデフォルトは `symbol` ですが、種類を明示的に指定することで可読性とパフォーマンスが向上します
2. **必要なフィールドのみをリクエストする** - レスポンスサイズを最小限に抑え、パフォーマンスを向上させるために、必要なロゴ種別のみをリクエストします
3. **大規模データセットには bulk-sync を使用する** - 複数のロゴを取得する場合は、ページネーション付きで `/logos/bulk-sync` エンドポイントを使用します
4. **レスポンスをキャッシュする** - ロゴの URL には有効期限のタイムスタンプが含まれるため、API 呼び出しを減らすよう適切にキャッシュします
5. **エラーを適切に処理する** - 常にレスポンス内の `ok` フィールドを確認し、エラーケースを適切に処理します
