Si tienes alguna pregunta, por favor [ponte en contacto con nosotros](mailto:partners@benzinga.com). Nos encanta ayudar.

## Acerca de esta documentación

Para nuestras APIs REST, esta documentación se basa en especificaciones OpenAPI. Puedes ver la especificación YAML sin procesar consultando el repositorio público de esta aplicación de documentación, [aquí](https://github.com/Benzinga/doc-site-mintlify).

Un cliente REST integrado se incluye directamente en la sección "Zona de pruebas" de cada página basada en OpenAPI. Una vez que hayas obtenido una clave de API, puedes probar llamadas sin salir del portal de documentación. Alternativamente, puedes probar llamadas con cualquier otro cliente REST con interfaz gráfica ([Postman](https://www.getpostman.com/), [Insomnia](https://insomnia.rest/), [Paw](https://paw.cloud/)) o con herramientas de CLI como cURL.

Finalmente, las especificaciones OpenAPI son compatibles con herramientas del ecosistema Swagger, por lo que incluso puedes [generar un cliente para el lenguaje de programación de tu elección](http://swagger.io/swagger-codegen/).

## Mejores prácticas

Recomendamos que los clientes utilicen nuestros clientes proporcionados por Benzinga siempre que sea posible. Muchos endpoints están disponibles en nuestra [biblioteca de Python](https://github.com/Benzinga/benzinga-python-client). También proporcionamos implementaciones de ejemplo y bibliotecas de cliente a través de nuestro [GitHub](https://github.com/benzinga) público.

Por último, recomendamos encarecidamente emplear deltas al usar las APIs de Calendar, Signals y News, especialmente cuando se utilizan para la ingestión en tiempo real (en lugar de la ingestión de datos históricos). Esto se puede hacer empleando el parámetro de consulta `parameters[updated]` para las APIs de Calendar y Signals, y el parámetro de consulta `updatedSince` para la API de News. El uso de estos parámetros garantiza una latencia mínima.
