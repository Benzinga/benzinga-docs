Si tienes alguna pregunta, por favor [ponte en contacto con nosotros](mailto:partners@benzinga.com). Nos encanta ayudar.

<div id="about-this-documentation">
  ## Acerca de esta Documentación
</div>

Para nuestras APIs REST, esta documentación está basada en especificaciones OpenAPI. Puedes ver la especificación YAML en bruto consultando el repositorio público de esta aplicación de documentación, [aquí](https://github.com/Benzinga/doc-site-mintlify).

Un cliente REST integrado se incluye directamente en la sección "Playground" de cada página basada en OpenAPI. Una vez que hayas obtenido una clave de API, puedes probar solicitudes sin salir del portal de documentación. Alternativamente, puedes probar solicitudes con cualquier otro cliente REST de interfaz gráfica ([Postman](https://www.getpostman.com/), [Insomnia](https://insomnia.rest/), [Paw](https://paw.cloud/)) o con herramientas de línea de comandos como cUrl1.

Por último, las especificaciones OpenAPI son compatibles con las herramientas del ecosistema Swagger, por lo que incluso puedes [generar un cliente para el lenguaje de programación de tu elección](http://swagger.io/swagger-codegen/).

<div id="acessing-the-data">
  ## Acceso a los datos
</div>

<Steps>
    <Step title="Inicia sesión en tu cuenta">
        Haz clic en "[Get Started](https://www.benzinga.com/apis/licensing/register)" para crear una cuenta.
    </Step>
    <Step title="Copia tu clave de API">
        Una vez que hayas iniciado sesión en el panel de usuario, copia tu clave de API.
    </Step>
    <Step title="¡Empieza a desarrollar!">
        La autenticación se realiza mediante una clave de API establecida como parámetro de consulta en la URL llamado "token".
    </Step>
</Steps>

<div id="best-practices">
  ## Mejores prácticas
</div>

Recomendamos que los clientes utilicen nuestros clientes proporcionados por Benzinga siempre que sea posible. Muchos endpoints están disponibles en nuestra [biblioteca de Python](https://github.com/Benzinga/benzinga-python-client). También proporcionamos implementaciones de ejemplo y bibliotecas de cliente a través de nuestro [Github](https://github.com/benzinga) público.

Por último, recomendamos encarecidamente emplear deltas al usar las APIs de Calendar, Signals y News, especialmente cuando se utilizan para la ingesta en tiempo real (en lugar de la ingesta de datos históricos). Esto se puede hacer utilizando el parámetro de consulta `parameters[updated]` para las APIs de Calendar y Signals, y el parámetro de consulta `updatedSince` para la API de News. El uso de estos parámetros garantiza una latencia mínima.
