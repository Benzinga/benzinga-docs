<div id="benzinga-apis">
  ## API de Benzinga
</div>

Desde los informes de resultados hasta la actividad de opciones, Benzinga se centra en la información que mueve el mercado. Ofrecemos un conjunto de canales centrados en noticias de última hora para que su empresa pueda ser la primera en ejecutar una operación. Ofrecemos dos formas de recibir nuestras noticias: mediante una API REST basada en solicitudes (&quot;pull&quot;) o mediante TCP con envío (&quot;push&quot;). Ambos ofrecen un acceso sin precedentes a los mercados.

Uno de los pilares de Benzinga es &quot;obsesionarnos con nuestros clientes, no con nuestra competencia&quot;.

Las relaciones sólidas son la fuerza motriz detrás de un negocio exitoso, y nuestro compromiso con ello está siempre en primer lugar.

Para comenzar con las API de Benzinga, envíenos un correo electrónico directamente a licensing@benzinga.com o llámenos al 877-440-9464. Responderemos en el plazo de un día hábil.

<div id="authentication">
  ## Autenticación
</div>

Incluye tu clave de API o token como el parámetro token en la cadena de consulta de la solicitud.

<div id="best-practices">
  ## Mejores prácticas
</div>

Sugerimos que los clientes utilicen, siempre que sea posible, los clientes oficiales proporcionados por Benzinga. Muchos endpoints están disponibles en nuestra [biblioteca de Python](/es/python-client). Proporcionamos implementaciones de referencia y bibliotecas de cliente a través de nuestro [GitHub](https://github.com/benzinga) público.

Por último, recomendamos encarecidamente emplear deltas al usar las APIs de Calendar, Signals y News, especialmente cuando se utilizan para la ingestión en tiempo real (en contraposición a la ingestión de datos históricos). Esto se puede hacer utilizando el parámetro de consulta `parameters[updated]` para las APIs de Calendar y Signals, y el parámetro de consulta `updatedSince` para la API de News. El uso de estos parámetros garantiza una latencia mínima.

<div id="max-offset">
  ## Desplazamiento máximo
</div>

La paginación de las APIs de News, Calendar y Signals no puede devolver conjuntos de resultados con más de 10.000 elementos. Es decir, si el tamaño de página es 1.000, no puedes solicitar la página 10.

Para solucionar esta limitación, debes ajustar el tamaño de la consulta. Por ejemplo, puedes restringirla a un rango de fechas específico o a un conjunto de tickers que incluya menos de 10.000 elementos. Después de hacerlo, puedes ajustar la consulta para incluir otro rango de fechas o conjunto de tickers que incluya menos de 10.000 elementos, combinar los conjuntos y repetir el proceso hasta obtener el conjunto de resultados completo que buscas.

<div id="return-format">
  ## Formato de retorno
</div>

El formato predeterminado para las APIs de Benzinga se especifica en las especificaciones individuales de cada API, pero será `JSON` a menos que se indique lo contrario.

Puedes solicitar `JSON` configurando el encabezado `accept` como `application/json`, o `XML` usando `application/xml` cuando este formato no esté en desuso.

**No se recomiendan nuevas integraciones que utilicen el formato de salida `XML`.**

<div id="example">
  ### Ejemplo
</div>

```JSON
curl -X GET \
'http://api.benzinga.com/api/v2/calendar/ratings?token=YOUR_TOKEN_HERE&ratings=Upgrades' \
-H 'accept: application/json'
```
