---
title: "Errores"
description: "Cómo responde la API de Benzinga cuando se producen errores."
---

Benzinga utiliza códigos de respuesta HTTP convencionales para indicar el éxito o el fallo de una solicitud de API.
En general: los códigos en el rango 2xx indican éxito.
Los códigos en el rango 4xx indican que la solicitud no pudo procesarse con la información proporcionada (por ejemplo, se omitió un parámetro requerido,
falló un cargo, etc.).
Los códigos en el rango 5xx indican un error en los servidores de Benzinga.

Algunos errores 4xx que se pueden manejar mediante programación incluyen
un código de error que explica brevemente el problema detectado.

<div id="parameters">
  ### Parámetros
</div>

<ResponseField name="type" type="string">
    El tipo de error devuelto.

    Uno de los siguientes valores: <code>invalid_request_error</code>, <code>not_found_error</code>, <code>creation_error</code>,
    <code>update_error</code>, <code>deletion_error</code>, <code>forbidden_error</code> o <code>authentication_error</code>.
</ResponseField>

<ResponseField name="message" type="string">
    Un mensaje legible para personas que proporciona más detalles sobre el error.
</ResponseField>

<ResponseField name="params" type="array">
    Si el error es específico de un parámetro, contendrá una lista de los parámetros que no eran válidos.
</ResponseField>
