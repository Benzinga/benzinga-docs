<div id="introduction">
  ## المقدمة
</div>

توفر واجهة برمجة التطبيقات API لشعارات Benzinga إمكانية الوصول إلى شعارات عالية الجودة للشركات والصناديق والعملات المشفرة. يمكنك البحث عن الشعارات باستخدام عدة أنواع من المعرّفات، بما في ذلك الرموز، وأرقام CIK، وأكواد رقم التعريف الدولي للأوراق المالية ISIN، وCUSIP، وFIGI، وغيرها.

<div id="available-endpoints">
  ## نقاط النهاية المتوفرة
</div>

<div id="logossearch">
  ### `/logos/search`
</div>

ابحث عن الشعارات باستخدام معرفات متعددة. تُرجِع عناوين url الخاصة بالشعارات وتفاصيل الأوراق المالية والبيانات الوصفية.

<div id="logossync">
  ### `/logos/sync`
</div>

نقطة نهاية للمزامنة المجمّعة لاسترجاع عدة شعارات مع دعم ترقيم الصفحات (pagination).

<div id="search-filters">
  ## عوامل تصفية البحث
</div>

تدعم واجهة برمجة التطبيقات API بحثًا مرنًا باستخدام معاملين رئيسيين:

- `search_keys` - قيمة المعرّف التي تريد البحث عنها
- `search_keys_type` - نوع المعرّف (اختياري؛ القيمة الافتراضية هي `symbol`)

<div id="1-filter-by-symbol">
  ### 1. التصفية حسب رمز التداول
</div>

ابحث عن الشعارات باستخدام رموز التداول للأسهم.

**المعاملات:**

* `search_keys`: رمز التداول (مثلًا: `AAPL`، `TSLA`، `MSFT`)
* `search_keys_type`: `symbol` (اختياري، وهو الإعداد الافتراضي)

**مثال:**

```
GET /api/v2/logos/search?search_keys=AAPL&search_keys_type=symbol&fields=logo_light,logo_dark
```


<div id="2-filter-by-cik">
  ### 2. التصفية حسب CIK
</div>

ابحث باستخدام مفتاح الفهرس المركزي (Central Index Key) الخاص بـ SEC. يدعم القيم مع الأصفار البادئة أو بدونها.

**المعاملات:**

* `search_keys`: رقم CIK (مثلًا: `320193` أو `0000320193`)
* `search_keys_type`: `cik`

**مثال:**

```
GET /api/v2/logos/search?search_keys=320193&search_keys_type=cik&fields=logo_light,logo_dark
```

**مثال على CIK مع أصفار بادئة:**

```
GET /api/v2/logos/search?search_keys=0000320193&search_keys_type=cik&fields=logo_light,logo_dark
```


<div id="3-filter-by-isin">
  ### 3. التصفية حسب رقم التعريف الدولي للأوراق المالية ISIN
</div>

ابحث باستخدام رقم التعريف الدولي للأوراق المالية ISIN.

**المعاملات:**

* `search_keys`: رمز رقم التعريف الدولي للأوراق المالية ISIN (مثلًا، `US0378331005`)
* `search_keys_type`: `isin`

**مثال:**

```
GET /api/v2/logos/search?search_keys=US0378331005&search_keys_type=isin&fields=logo_light,logo_dark
```


<div id="4-filter-by-figi-share-class">
  ### 4. تصفية البحث حسب فئة أسهم FIGI
</div>

ابحث باستخدام المعرّف العالمي للأداة المالية (FIGI) من Bloomberg لفئة الأسهم.

**المعاملات:**

* `search_keys`: معرّف فئة أسهم FIGI (مثلًا: `BBG001SQKGD7`)
* `search_keys_type`: `figi_share_class`

**مثال:**

```
GET /api/v2/logos/search?search_keys=BBG001SQKGD7&search_keys_type=figi_share_class&fields=logo_light,logo_dark
```


<div id="5-filter-by-cusip">
  ### 5. التصفية حسب CUSIP
</div>

ابحث باستخدام رقم لجنة إجراءات توحيد تعريف الأوراق المالية (Committee on Uniform Securities Identification Procedures).

**المعلمات:**

* `search_keys`: رقم CUSIP (مثلًا: `88160R101`)
* `search_keys_type`: `cusip`

**مثال:**

```
GET /api/v2/logos/search?search_keys=88160R101&search_keys_type=cusip&fields=logo_light,logo_dark
```


<div id="6-filter-by-symbol-with-currency-country">
  ### 6. التصفية حسب الرمز مع العملة والدولة
</div>

قيِّد الرموز برموز الدولة أو العملة باستخدام النقطتين (`:`) أو النقطة (`.`) كفاصل.

**المعاملات:**

* `search_keys`: الرمز مع المحدِّد (مثل: `AAPL:US`، `AAPL:USD`، `AAPL.US`)
* `search_keys_type`: `symbol`

**أمثلة:**

```
GET /api/v2/logos/search?search_keys=AAPL:US&search_keys_type=symbol&fields=logo_light,logo_dark
GET /api/v2/logos/search?search_keys=AAPL:USD&search_keys_type=symbol&fields=logo_light,logo_dark
GET /api/v2/logos/search?search_keys=AAPL.US&search_keys_type=symbol&fields=logo_light,logo_dark
```


<div id="7-filter-by-exchange-and-symbol">
  ### 7. التصفية حسب البورصة والرمز
</div>

ابحث عن طريق الجمع بين رمز الـ exchange مع الرمز. يدعم كلًا من تنسيق Bloomberg (قديم) وتنسيق LSEG الحالي.

**مطابقة بأسلوب Bloomberg (قديم):**

* `search_keys`: exchange والرمز (مثلًا: `NASDAQ:AAPL`)
* `search_keys_type`: `symbol`

**رمز LSEG مكوّن من 3 أحرف (حالي):**

* `search_keys`: رمز الـ exchange والرمز (مثلًا: `NSQ:AAPL`)
* `search_keys_type`: `symbol`

**أمثلة:**

```
GET /api/v2/logos/search?search_keys=NASDAQ:AAPL&search_keys_type=symbol&fields=logo_light,logo_dark
GET /api/v2/logos/search?search_keys=NSQ:AAPL&search_keys_type=symbol&fields=logo_light,logo_dark
```


<div id="8-filter-by-iso-mic-and-symbol">
  ### 8. التصفية حسب ISO MIC والرمز
</div>

ابحث باستخدام رمز معرّف السوق وفق معيار ISO (ISO Market Identifier Code) مع الرمز.

**المعلمات:**

* `search_keys`: رمز MIC والرمز (على سبيل المثال، `XYNS:AAPL`)
* `search_keys_type`: `symbol`

**مثال:**

```
GET /api/v2/logos/search?search_keys=XYNS:AAPL&search_keys_type=symbol&fields=logo_light,logo_dark
```


<div id="9-crypto-logo-filters">
  ### 9. عوامل تصفية شعارات العملات المشفّرة
</div>

ابحث عن شعارات العملات المشفّرة باستخدام تنسيقات متعدّدة. تتعرّف واجهة برمجة التطبيقات API تلقائيًا على تنسيقات العملات المشفّرة حتى إذا لم يتم تحديد `search_keys_type`.

**التنسيقات المدعومة:**

* `CRYPTO:BTC`
* `CRYPTO/BTC`
* `$BTC`
* `BTC/USD`

**أمثلة:**

```
GET /api/v2/logos/search?search_keys=CRYPTO:BTC&fields=logo_light,logo_dark
GET /api/v2/logos/search?search_keys=CRYPTO/BTC&fields=logo_light,logo_dark
GET /api/v2/logos/search?search_keys=$BTC&fields=logo_light,logo_dark
GET /api/v2/logos/search?search_keys=BTC/USD&fields=logo_light,logo_dark
```


<div id="available-fields">
  ## الحقول المتاحة
</div>

يمكنك طلب أنواع محددة من الشعارات باستخدام معامل `fields`. تتضمن الحقول المتاحة:

* `logo_light` - شعار للوضع الفاتح (PNG)
* `logo_dark` - شعار للوضع الداكن (PNG)
* `logo_vector_light` - شعار متجه للوضع الفاتح (SVG)
* `logo_vector_dark` - شعار متجه للوضع الداكن (SVG)
* `mark_light` - علامة/أيقونة للوضع الفاتح (PNG)
* `mark_dark` - علامة/أيقونة للوضع الداكن (PNG)
* `mark_vector_light` - علامة متجهة للوضع الفاتح (SVG)
* `mark_vector_dark` - علامة متجهة للوضع الداكن (SVG)
* `mark_composite_light` - علامة مركّبة للوضع الفاتح (PNG)
* `mark_composite_dark` - علامة مركّبة للوضع الداكن (PNG)
* `mark_vector_composite_light` - علامة متجهة مركّبة للوضع الفاتح (SVG)
* `mark_vector_composite_dark` - علامة متجهة مركّبة للوضع الداكن (SVG)

**مثال على طلب عدة حقول:**

```
GET /api/v2/logos/search?search_keys=TSLA&fields=logo_light,logo_dark,mark_vector_composite_light
```


<div id="additional-parameters">
  ## معلمات إضافية
</div>

<div id="image-customization">
  ### تخصيص الصورة
</div>

- `scale` - تغيير حجم الصورة (مثلًا، `100x100`، `300x300`)
- `max_width` - الحد الأقصى لعرض الصورة المُعادة
- `composite_radius` - نصف قطر حواف الصور المركّبة (عدد صحيح، النطاق `0-50`)
- `composite_auto` - إنشاء الصور المركّبة تلقائيًا (قيمة منطقية)

<div id="pagination-bulk-sync-only">
  ### ترقيم الصفحات (bulk-sync فقط)
</div>

- `page` - رقم الصفحة (عدد صحيح)
- `pagesize` - عدد النتائج لكل صفحة (عدد صحيح)
- `updated_since` - تصفية النتائج حسب طابع زمن آخر تحديث (صيغة ISO 8601)

<div id="security-details">
  ### تفاصيل الأوراق المالية
</div>

- `securities` - تضمين معلومات تفصيلية عن الأوراق المالية في الاستجابة (قيمة منطقية)

<div id="authentication">
  ## المصادقة
</div>

تتطلب جميع نقاط نهاية واجهة برمجة التطبيقات API الخاصة بـ Logos مصادقة باستخدام مفتاح واجهة برمجة التطبيقات API. قم بتضمين رمز الوصول الخاص بك كمعامل استعلام:

```
?token=YOUR_API_KEY
```


<div id="best-practices">
  ## أفضل الممارسات
</div>

1. **حدِّد search_keys_type** - بالرغم من أن واجهة برمجة التطبيقات API تستخدم القيمة الافتراضية `symbol`، فإن تحديد النوع صراحةً يحسِّن الوضوح والأداء
2. **اطلب الحقول اللازمة فقط** - اطلب فقط أنواع الشعارات التي تحتاجها لتقليل حجم الاستجابة وتحسين الأداء
3. **استخدم bulk-sync لمجموعات البيانات الكبيرة** - عند استرجاع عدة شعارات، استخدم نقطة النهاية `/logos/bulk-sync` مع ترقيم الصفحات
4. **خزِّن الاستجابات مؤقتًا** - عناوين URL الخاصة بالشعارات تتضمن طوابع زمنية لانتهاء الصلاحية؛ خزِّنها بالشكل المناسب لتقليل عدد استدعاءات واجهة برمجة التطبيقات API
5. **تعامَل مع الأخطاء بسلاسة** - تحقَّق دائمًا من الحقل `ok` في الاستجابات وتعامل مع حالات الخطأ بالشكل المناسب

<div id="rate-limits">
  ## حدود معدل الطلبات
</div>

راجع خطة اشتراكك في واجهة برمجة التطبيقات API للاطلاع على تفاصيل حدود معدل الطلبات. تواصل مع فريق الدعم إذا كنت بحاجة إلى حدود معدل طلبات أعلى لتطبيقك.
