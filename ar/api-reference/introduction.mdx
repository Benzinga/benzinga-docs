---
title: 'المقدمة'
description: 'مقدمة موجزة عن واجهة برمجة التطبيقات API الخاصة بـ Benzinga.'
---

مرحبًا بك في توثيق واجهة برمجة التطبيقات API الخاصة بـ Benzinga. سيساعدك هذا الدليل على فهم كيفية التفاعل مع واجهة برمجة التطبيقات API الخاصة بـ Benzinga لإنشاء واسترجاع وتحديث وحذف موارد Benzinga عبر طلبات HTTP.

<div id="base-url">
  ## عنوان URL الأساسي
</div>

جميع عناوين URL المذكورة في التوثيق تشترك في عنوان URL أساسي واحد هو:

```
https://api.benzinga.com
```


<div id="authentication">
  ## المصادقة
</div>

تستخدم واجهة برمجة التطبيقات API من Benzinga مصادقة من نوع Bearer. عند إرسال الطلبات، يجب عليك تضمين رمز الوصول الخاص بواجهة برمجة التطبيقات API في معامِل الاستعلام `token`
بصيغة `token: <token>`. لمزيد من التفاصيل، يُرجى الرجوع إلى صفحة [المصادقة](./authentication).

<div id="errors">
  ## الأخطاء
</div>

عند حدوث خطأ، تستجيب واجهة برمجة التطبيقات API من Benzinga برمز حالة HTTP قياسي وكائن JSON يحتوي على مزيد من
التفاصيل حول الخطأ. لمزيد من المعلومات، راجع صفحة [الأخطاء](./errors).

<div id="rate-limiting">
  ## تحديد المعدل
</div>

يرجى ملاحظة أن واجهة برمجة التطبيقات API من Benzinga تفرض حدودًا على معدل الطلبات لمنع إساءة الاستخدام وضمان استقرار الخدمة. في حال تجاوزت هذه الحدود،
سيتم تقييد طلباتك وستتلقى استجابة `429 Too Many Requests`.

---

نأمل أن يساعدك هذا الدليل على البدء في استخدام واجهة برمجة التطبيقات API من Benzinga. إذا كانت لديك أي أسئلة، فلا تتردد في التواصل
مع فريق الدعم لدينا.
