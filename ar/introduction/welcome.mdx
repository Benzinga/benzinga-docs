في بوابة توثيق المطورين ضمن [Benzinga Cloud](https://cloud.benzinga.com/) ستجد توثيق واجهات برمجة التطبيقات API لبياناتنا الإخبارية وبيانات السوق وبيانات الشركات. إذا كانت لديك أي أسئلة، فيرجى [التواصل معنا](mailto:partners@benzinga.com). يسعدنا دائمًا تقديم المساعدة.

<div id="about-this-documentation">
  ## حول هذا التوثيق
</div>

بالنسبة لواجهات برمجة تطبيقات REST، يستند هذا التوثيق إلى مواصفات OpenAPI. يمكنك عرض مواصفة YAML الخام من خلال الاطلاع على المستودع العام لتطبيق التوثيق هذا، من [هنا](https://github.com/Benzinga/doc-site-mintlify).

يوجد عميل REST مدمج مباشرة في قسم "Playground" لكل صفحة تعتمد على OpenAPI. بمجرد حصولك على مفتاح API، يمكنك اختبار الاستدعاءات دون مغادرة بوابة التوثيق. بدلاً من ذلك، يمكنك اختبار الاستدعاءات باستخدام أي عميل REST بواجهة مستخدم رسومية أخرى ([Postman](https://www.getpostman.com/)، [Insomnia](https://insomnia.rest/)، [Paw](https://paw.cloud/)) أو أدوات سطر الأوامر مثل cUrl1.

أخيرًا، تكون مواصفات OpenAPI متوافقة مع الأدوات في منظومة Swagger، بحيث يمكنك حتى [توليد عميل برمجي للغة البرمجة التي تختارها](http://swagger.io/swagger-codegen/).

<Info>
    التوثيق لدينا مدعوم بالذكاء الاصطناعي. جرِّب طرح سؤال باستخدام Ctrl + K (على الكمبيوتر الشخصي) أو Cmd + K (على أجهزة Mac).
</Info>

<div id="acessing-the-data">
  ## الوصول إلى البيانات
</div>

<Steps>
    <Step title="تسجيل الدخول إلى حسابك">
        انقر على "[Get Started](https://www.benzinga.com/apis/licensing/register)" لإنشاء حساب.
    </Step>
    <Step title="انسخ مفتاح واجهة برمجة التطبيقات API الخاص بك">
        بعد تسجيل الدخول إلى لوحة تحكم المستخدم، انسخ مفتاح واجهة برمجة التطبيقات API الخاص بك.
    </Step>
    <Step title="ابدأ التطوير!">
        تتم المصادقة باستخدام مفتاح واجهة برمجة التطبيقات API المُمرَّر كمعامل استعلام في عنوان URL باسم "token".
    </Step>
</Steps>

<div id="best-practices">
  ## أفضل الممارسات
</div>

نوصي بأن يستخدم العملاء مكتبات العميل (clients) التي توفّرها Benzinga كلما كان ذلك ممكنًا. العديد من نقاط النهاية متاحة في [مكتبة Python](https://github.com/Benzinga/benzinga-python-client) الخاصة بنا، كما نقدّم تطبيقات توضيحية ومكتبات عملاء عبر [Github](https://github.com/benzinga) العام الخاص بنا.

أخيرًا، نوصي بشدة باستخدام التغييرات (deltas) عند استخدام calendar وSignals وNews ضمن واجهة برمجة التطبيقات API، خصوصًا عند استخدامها لإدخال البيانات الآني (real-time ingestion) بدلًا من إدخال البيانات التاريخية. يمكن تنفيذ ذلك باستخدام معامل الاستعلام `parameters[updated]` مع calendar وSignals، ومعامل الاستعلام `updatedSince` مع News. يضمن استخدام هذه المعاملات تقليل زمن الاستجابة إلى الحد الأدنى.
