<div id="message-structure">
  ## بنية الرسالة
</div>

```json
{
  "id": 49917328,
  "title": "تنبيه الخيارات: GOOG $210 Calls",
  "body": "",
  "authors": [{"name": "Benzinga Newsdesk"}],
  "published": "Wed Jan 14 2026 18:47:30 GMT+0000 (UTC)",
  "updated": "Wed Jan 14 2026 18:47:30 GMT+0000 (UTC)",
  "channels": ["Options"],
  "tickers": [{"name": "GOOG", "primary": false, "sentiment": 0}],
  "status": "Published",
  "link": null
}
```


<div id="field-reference">
  ## مرجع الحقول
</div>

| الحقل | النوع | الوصف |
|-------|------|-------------|
| `id` | عدد صحيح | معرّف فريد لعنصر المحتوى |
| `title` | سلسلة نصية | عنوان المحتوى أو العنوان الرئيسي |
| `body` | سلسلة نصية | نص المحتوى الكامل (منسَّق بصيغة HTML عند توفره) |
| `authors` | مصفوفة | قائمة كائنات المؤلفين التي تتضمّن خاصية `name` |
| `published` | سلسلة نصية | الطابع الزمني للنشر الأصلي (UTC) |
| `updated` | سلسلة نصية | الطابع الزمني لآخر تحديث (UTC) |
| `channels` | مصفوفة | فئات المحتوى (مثلًا: "الخيارات"، "البيانات الصحفية") |
| `tickers` | مصفوفة | رموز الأسهم المرتبطة مع بياناتها الوصفية |
| `status` | سلسلة نصية | حالة النشر (عادةً "Published") |
| `link` | سلسلة نصية/null | عنوان URL للمقال الكامل على Benzinga (إن وُجد) |

<div id="ticker-object">
  ### كائن رمز التداول
</div>

```json
{
  "name": "GOOG",
  "primary": false,
  "sentiment": 0
}
```

| الحقل       | النوع   | الوصف                                         |
| ----------- | ------- | --------------------------------------------- |
| `name`      | String  | رمز تداول السهم                               |
| `primary`   | Boolean | ما إذا كان هذا هو رمز التداول الأساسي للمحتوى |
| `sentiment` | Integer | مؤشر المعنويات (0 = حيادي)                    |

<div id="processing-tips">
  ## نصائح للمعالجة
</div>

<Tip>
تُرسل الرسائل بصيغة JSON مفصول بأسطر جديدة (newline-delimited). يحتوي كل سطر على كائن JSON كامل.
</Tip>

<div id="best-practices">
  ### أفضل الممارسات
</div>

1. **تحليل كل سطر**: يمثّل كل سطر جديد رسالة JSON كاملة
2. **معالجة HTML في الحقل `body`**: قد يحتوي الحقل `body` على كيانات وعناصر HTML
3. **التحقق من التحديثات**: قارِن بين الطوابع الزمنية `published` و`updated` لتحديد التعديلات
4. **التصفية حسب القناة**: استخدم مصفوفة `channels` لتصفية أنواع معيّنة من المحتوى
5. **التتبّع باستخدام المعرّف**: استخدم الحقل `id` كمعرّف فريد لإزالة التكرار
