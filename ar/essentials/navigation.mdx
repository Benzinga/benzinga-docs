---

قائمة التنقّل هي قائمة الروابط الموجودة في كل موقع إلكتروني.

على الأرجح ستحتاج إلى تحديث `docs.json` في كل مرة تضيف فيها صفحة جديدة. الصفحات لا تظهر تلقائيًا.

<div id="navigation-syntax">
  ## بنية التنقل
</div>

بنية التنقل لدينا تكرارية (recursive)، مما يعني أنه يمكنك إنشاء مجموعات تنقل متداخلة. لست بحاجة إلى تضمين `.mdx` في أسماء الصفحات. علامات التبويب تحتوي المجموعات، والمجموعات تحتوي الصفحات.

<CodeGroup>

```json تنقل عادي
"navigation": {
  "tabs": [
    {
      "tab": "التوثيق",
      "groups": [
        {
          "group": "البدء",
          "pages": ["quickstart"]
        }
      ]
    }
  ]
}
```

```json تنقل متداخل
"navigation": {
  "tabs": [
    {
      "tab": "التوثيق",
      "groups": [
        {
          "group": "البدء",
          "pages": [
            "quickstart",
            {
              "group": "صفحات مرجعية متداخلة",
              "pages": ["nested-reference-page"]
            }
          ]
        }
      ]
    }
  ]
}
```

</CodeGroup>

<div id="folders">
  ## المجلدات
</div>

ما عليك سوى وضع ملفات MDX الخاصة بك في مجلدات وتحديث المسارات في `docs.json`.

على سبيل المثال، لإنشاء صفحة على `https://yoursite.com/your-folder/your-page` ستنشئ مجلدًا باسم `your-folder` يحتوي على ملف MDX باسم `your-page.mdx`.

<Warning>
  لا يمكنك استخدام `api` كاسم لمجلد إلا إذا قمت بتعشيقه داخل مجلد آخر. يستخدم Mintlify إطار Next.js الذي يحجز مجلد `api` في المستوى الأعلى لاستدعاءات الخادم الداخلية. سيكون اسم مجلد مثل `api-reference` مقبولًا.
</Warning>

```json Navigation With Folder
"navigation": {
  "tabs": [
    {
      "tab": "التوثيق",
      "groups": [
        {
          "group": "Group Name",
          "pages": ["your-folder/your-page"]
        }
      ]
    }
  ]
}
```


<div id="hidden-pages">
  ## الصفحات المخفية
</div>

ملفات MDX غير المدرجة في `docs.json` لن تظهر في الشريط الجانبي، لكن يمكن الوصول إليها من خلال شريط البحث أو عبر الربط المباشر إليها.
