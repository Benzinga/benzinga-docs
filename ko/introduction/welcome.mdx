[Benzinga Cloud](https://cloud.benzinga.com/) 개발자 문서 포털에서는 당사의 뉴스, 시장, 기업 데이터 API에 대한 문서를 확인할 수 있습니다. 문의 사항이 있으시면 언제든지 [연락해 주세요](mailto:partners@benzinga.com). 기꺼이 도와드리겠습니다.

<div id="about-this-documentation">
  ## 이 문서에 대하여
</div>

REST API 문서는 OpenAPI 명세를 기반으로 합니다. 이 문서 애플리케이션의 공개 저장소에서 원본 YAML 명세를 [여기](https://github.com/Benzinga/doc-site-mintlify)에서 확인할 수 있습니다.

통합 REST 클라이언트는 OpenAPI로 구동되는 각 페이지의 "Playground" 섹션에 직접 내장되어 있습니다. API 키를 발급받은 후에는 문서 포털을 벗어나지 않고도 호출을 테스트할 수 있습니다. 또는 다른 GUI REST 클라이언트([Postman](https://www.getpostman.com/), [Insomnia](https://insomnia.rest/), [Paw](https://paw.cloud/))나 cUrl1 같은 CLI 도구를 사용해 호출을 테스트할 수도 있습니다.

마지막으로, OpenAPI 명세는 Swagger 에코시스템의 도구들과 호환되므로, 원하는 프로그래밍 언어용 [클라이언트를 생성](http://swagger.io/swagger-codegen/)할 수도 있습니다.

<Info>

</Info>

</div>

<Steps>

    </Step>
</Steps>

<div id="best-practices">
  ## 모범 사례
</div>

가능한 경우 Benzinga에서 제공하는 클라이언트를 사용하는 것을 권장합니다. 많은 엔드포인트는 [Python 라이브러리](https://github.com/Benzinga/benzinga-python-client)에서 사용할 수 있습니다. 또한 공개 [GitHub](https://github.com/benzinga)를 통해 예제 구현 및 클라이언트 라이브러리를 제공합니다.

마지막으로, 특히 실시간 데이터 수집(과거 데이터 수집과 대비) 시 Calendar, Signals, News API를 사용할 때는 델타(delta) 방식을 활용할 것을 강력히 권장합니다. Calendar 및 Signals API에서는 `parameters[updated]` 쿼리 파라미터를, News API에서는 `updatedSince` 쿼리 파라미터를 사용해 이를 구현할 수 있습니다. 이러한 파라미터를 사용하면 지연 시간을 최소화할 수 있습니다.
