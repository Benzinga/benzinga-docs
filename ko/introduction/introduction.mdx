## Benzinga API

실적 보고서부터 옵션 거래까지, Benzinga는 시장을 움직이는 정보에 집중합니다. 우리는 귀사의 firm이 누구보다 먼저 거래를 체결할 수 있도록 속보에 특화된 다양한 채널 제품군을 제공합니다. 당사의 뉴스는 pull 기반 REST API 또는 push 기반 TCP를 통해 받아보실 수 있습니다. 두 방식 모두 전례 없는 수준의 시장 접근성을 제공합니다.

Benzinga의 핵심 원칙 중 하나는 “경쟁사가 아니라 우리 고객에게 집착하라”입니다.

신뢰할 수 있는 긴밀한 관계는 성공적인 비즈니스를 이끄는 원동력이며, 이에 대한 우리의 헌신은 언제나 최우선입니다.

Benzinga API를 사용해 보려면 licensing@benzinga.com으로 이메일을 보내시거나 877-440-9464로 전화해 주십시오. 영업일 기준 1일 이내에 회신드립니다.

## 인증

요청의 쿼리 문자열에서 token 파라미터 값으로 API 키 또는 token을 포함하세요.

## 모범 사례

가능한 경우 Benzinga에서 제공하는 클라이언트를 사용할 것을 권장합니다. 많은 엔드포인트가 [Python 라이브러리](/python-client)에 포함되어 있습니다. 또한 공개 [GitHub](https://github.com/benzinga)를 통해 레퍼런스 구현과 클라이언트 라이브러리를 제공합니다.

마지막으로, Calendar, Signals, News API를 사용할 때는 특히 실시간 수집(과거 데이터 수집과 대비되는 경우)에 deltas 방식을 사용하는 것을 강력히 권장합니다. Calendar 및 Signals API에서는 `parameters[updated]` 쿼리 파라미터를, News API에서는 `updatedSince` 쿼리 파라미터를 사용하여 이를 구현할 수 있습니다. 이러한 파라미터를 사용하면 지연 시간을 최소화할 수 있습니다.

## 최대 오프셋

News, Calendar, Signals API의 페이지네이션은 한 번에 10,000개를 초과하는 결과 집합을 반환할 수 없습니다. 즉, 페이지 크기(page size)가 1,000인 경우 10페이지를 요청할 수 없습니다.

이 제한을 우회하려면 쿼리 범위를 조정해야 합니다. 예를 들어, 10,000개 미만의 항목만 포함되도록 특정 날짜 범위나 티커 집합으로 제한할 수 있습니다. 그런 다음 10,000개 미만의 항목을 포함하는 다른 날짜 범위나 티커 집합으로 쿼리를 다시 조정하고, 각 결과 집합을 결합한 뒤, 원하는 전체 결과 집합을 얻을 때까지 이 과정을 반복하면 됩니다.

## 반환 형식

Benzinga API의 기본 형식은 각 API 사양에서 정의되며, 별도로 명시되지 않는 한 `JSON`입니다.

해당 형식이 사용 중단(deprecated) 상태가 아닌 경우, `accept` 헤더를 `application/json`으로 설정해 `JSON`을 요청하거나 `application/xml`을 사용해 `XML`을 요청할 수 있습니다.

**`XML` 출력 형식을 사용하는 신규 연동은 권장되지 않습니다.**

### 예시

```JSON
curl -X GET \
'http://api.benzinga.com/api/v2/calendar/ratings?token=YOUR_TOKEN_HERE&ratings=Upgrades' \
-H 'accept: application/json'
```
