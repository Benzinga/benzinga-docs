여기 [Benzinga Cloud](https://cloud.benzinga.com/) 개발자 문서 포털에서는 뉴스, 시장, 기업 데이터 API에 대한 문서를 확인할 수 있습니다. 질문이 있으시면 언제든지 [문의해 주세요](mailto:partners@benzinga.com). 기꺼이 도와드리겠습니다.

<div id="about-this-documentation">
  ## 이 문서에 대해
</div>

REST API의 경우, 이 문서는 OpenAPI 사양을 기반으로 작성되었습니다. 이 문서 애플리케이션의 퍼블릭 리포지토리에서 원본 YAML 사양을 [여기](https://github.com/Benzinga/doc-site-mintlify)에서 확인할 수 있습니다.

통합 REST 클라이언트는 OpenAPI로 구동되는 각 페이지의 "Playground" 섹션에 직접 내장되어 있습니다. API 키를 발급받으면 문서 포털을 벗어나지 않고도 API 호출을 테스트할 수 있습니다. 또는 다른 GUI REST 클라이언트([Postman](https://www.getpostman.com/), [Insomnia](https://insomnia.rest/), [Paw](https://paw.cloud/))나 cUrl1 같은 CLI 도구로 호출을 테스트할 수도 있습니다.

마지막으로, OpenAPI 사양은 Swagger 에코시스템의 도구와 호환되므로, [선호하는 프로그래밍 언어용 클라이언트를 생성](http://swagger.io/swagger-codegen/)할 수도 있습니다.

<Info>
    이 문서에는 AI 기능이 포함되어 있습니다. PC에서는 Ctrl + K, Mac에서는 Cmd + K를 눌러 질문을 입력해 보세요.
</Info>

    </Step>
</Steps>

<div id="best-practices">
  ## 모범 사례
</div>

가능한 경우 Benzinga에서 제공하는 클라이언트를 사용하는 것을 권장합니다. 많은 엔드포인트가 당사의 [Python 라이브러리](https://github.com/Benzinga/benzinga-python-client)에 포함되어 있습니다. 또한 공개 [GitHub](https://github.com/benzinga)를 통해 예제 구현과 클라이언트 라이브러리를 제공합니다.

마지막으로 Calendar, Signals, News API를 사용할 때, 특히 실시간 데이터 수집(과거 데이터 수집이 아닌)을 하는 경우에는 델타 처리(deltas) 방식을 사용하는 것을 강력히 권장합니다. Calendar 및 Signals API에서는 `parameters[updated]` 쿼리 파라미터를, News API에서는 `updatedSince` 쿼리 파라미터를 사용하여 이를 구현할 수 있습니다. 이러한 파라미터를 사용하면 지연 시간을 최소화할 수 있습니다.
