---
title: '소개'
description: 'Benzinga API에 대한 간단한 소개입니다.'
---

Benzinga API 문서에 오신 것을 환영합니다. 이 가이드는 HTTP 요청을 통해 Benzinga 리소스를 생성, 조회, 수정 및 삭제하기 위해 Benzinga API를 사용하는 방법을 이해하는 데 도움이 됩니다.

<div id="base-url">
  ## 기본 URL
</div>

문서에서 참조되는 모든 url은 다음 기본 url을 기반으로 합니다:

```
https://api.benzinga.com
```


<div id="authentication">
  ## 인증
</div>

Benzinga API는 Bearer 인증을 사용합니다. 요청을 보낼 때는 `token` 쿼리 매개변수에 `token: <token>` 형식으로 API 토큰을 포함해야 합니다. 자세한 내용은 [인증](./authentication) 페이지를 참고하세요.

<div id="errors">
  ## 오류
</div>

오류가 발생하면 Benzinga API는 통상적인 HTTP 응답 코드와 오류에 대한 추가 세부 정보가 담긴 JSON 객체를 반환합니다. 자세한 내용은 [오류](./errors) 페이지를 참조하세요.

<div id="rate-limiting">
  ## Rate Limiting
</div>

Benzinga API는 오남용을 방지하고 서비스 안정성을 보장하기 위해 요청률 제한이 적용됩니다. 이 제한을 초과하면
요청에 제한(Throttling)이 걸리며 `429 Too Many Requests` 응답을 받게 됩니다.
